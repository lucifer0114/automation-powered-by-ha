# v1.0.4 Benchmark Notes

## Goal
Measure whether v1.0.4 actually shortens end-to-end Weibo evidence generation time without relaxing the current quality bar.

## Fixed test conditions
- Use the persistent logged-in Playwright profile already used by production runs.
- Use a user-supplied Weibo URL; do not swap to an easier post.
- Keep the same target comment text for retries on the same smoke run.
- Record whether the run used:
  - `--submit`
  - `--capture-only`
  - `--capture-first-submit-fallback`

## Commands

### Submit + capture smoke run
```bash
python3 script/weibo_manual_comment_flow.py \
  --url 'PASTE_WEIBO_URL' \
  --comment 'PASTE_COMMENT' \
  --submit \
  --like
```

### Capture-only smoke run
```bash
python3 script/weibo_manual_comment_flow.py \
  --url 'PASTE_WEIBO_URL' \
  --comment 'PASTE_COMMENT' \
  --capture-only
```

## What to record
Each run now prints a machine-readable line:
```text
TIMINGS={...}
```

Capture these fields for comparison:
- `open_url_ms`
- `initial_ready_ms`
- `open_composer_ms`
- `fill_comment_ms`
- `submit_wait_ms`
- `locate_comment_ms`
- `capture_artifacts_ms`
- `total_ms`

Also note:
- `PAGE_READY=` value
- `POST_SUBMIT_READY=` value when submit mode is used
- final screenshot path used for delivery

## Release gate
Treat v1.0.4 as acceptable only if all are true:
- screenshot still shows the publishing account/header plus the full post body
- unrelated comments remain minimal
- red box is exact
- no new false negatives in comment detection, including the submit-path composer probe
- measured `total_ms` is lower than the v1.0.3 baseline on comparable real runs
